# CDN-FRAMEWORK GitHub Configuration

## Branch strategy
- `main`: Production-ready code
- `develop`: Development branch for new features
- `feature/*`: Feature branches
- `bugfix/*`: Bug fix branches

## Pull Request guidelines
1. Usa branch feature/bugfix
2. Descrivi le modifiche nel PR
3. Test prima di aprire PR
4. Rispondi ai commenti

## Commit messages
- `feat:` Nuova feature
- `fix:` Bug fix
- `docs:` Documentazione
- `test:` Test
- `refactor:` Refactoring

Formato: `type(scope): description`
Esempio: `feat(nmap): add XML parsing support`

## Code style
- Python 3.8+
- PEP 8 compliance
- 4 spaces indentation
- Type hints preferred

## Issues
Usa template issue per:
- Bug reports
- Feature requests
- Documentation improvements
